# 医药知识图谱 → AI智能问答Agent · 项目重构书

## 一、项目概述

### 1.1 项目背景

传统医药问答系统主要依赖关键词匹配和规则模板，存在以下痛点：

- **语义理解弱**：用户问“感冒了吃什么药”和“上呼吸道感染用药”无法关联
- **回答生硬**：模板拼接式回答，缺乏自然语言流畅性
- **可维护性差**：新增问题类型需要修改代码规则
- **无法展示**：本地命令行运行，不能给面试官直接体验

### 1.2 项目目标

将开源医药知识图谱项目（4.4万实体、30万关系）重构为**可公网访问的AI智能问答Agent**，实现自然语言问句到知识图谱查询的完整链路。

### 1.3 项目成果

| 维度     | 重构前                 | 重构后                       |
| :------- | :--------------------- | :--------------------------- |
| 交互方式 | 命令行                 | Web应用（公网可访问）        |
| 意图识别 | 规则匹配（18类硬编码） | LLM意图识别 + 代码规则兜底   |
| 答案生成 | 模板拼接               | 大模型自然语言生成           |
| 技术栈   | Python + Neo4J         | Dify + Neo4J + Qwen + Docker |
| 可展示性 | 本地运行               | 公网链接，随时体验           |

### 3.2 Neo4J API 封装

将知识图谱查询能力封装为 RESTful API：

python

```
# neo4j_api.py 核心代码
from flask import Flask, request, jsonify
from py2neo import Graph

app = Flask(__name__)
graph = Graph("bolt://host.docker.internal:7687", auth=("neo4j", "password"))

QUERY_TEMPLATES = {
    "disease_symptom": "MATCH (d:Disease {name: $entity})-[:has_symptom]->(s) RETURN s.name",
    "disease_drug": "MATCH (d:Disease {name: $entity})-[:recommand_drug]->(dr) RETURN dr.name",
    # ... 18类查询模板
}

@app.route('/query', methods=['POST'])
def query_neo4j():
    data = request.get_json()
    query = QUERY_TEMPLATES[data['type']]
    results = graph.run(query, parameters={"entity": data['entity']}).data()
    return jsonify({"results": [r['name'] for r in results if r.get('name')]})
```

### 3.4 Dify 工作流设计

**工作流节点**：

| 节点     | 类型   | 功能                           |
| :------- | :----- | :----------------------------- |
| 开始节点 | 输入   | 接收用户问题                   |
| 意图识别 | LLM    | 判断问题类型（18类），提取实体 |
| 代码节点 | Python | 解析LLM输出，规则兜底          |
| 工具节点 | API    | 调用Neo4J API查询知识图谱      |
| 答案生成 | LLM    | 基于检索结果生成自然回答       |
| 输出节点 | 输出   | 返回最终答案                   |

**意图识别Prompt**：

text

```
你是一个医药问答意图识别专家。判断用户问题属于以下哪一类：
- disease_symptom: 查询疾病症状
- disease_drug: 查询疾病用药
- disease_check: 查询检查项目
- ...

用户问题：{{question}}
输出JSON格式：{"type": "类型", "entity": "实体名称"}
```



**答案生成Prompt**：

text

```
你是一个专业的医药问答助手。基于以下查询结果回答用户问题。

查询结果：{{results}}
用户问题：{{question}}

要求：
1. 只基于查询结果回答，不要编造
2. 回答简洁、专业、易懂
3. 末尾加上“本回答仅供参考，具体请咨询医生”
```

## 五、遇到的问题与解决方案

| 问题                   | 解决方案                                  |
| :--------------------- | :---------------------------------------- |
| Neo4J热点写问题        | 设计“盐值+itemId”组合行键，预分区分散数据 |
| Dify工作流变量传递失败 | 使用代码节点解析JSON，规则兜底意图识别    |
| Docker容器间通信       | 加入同一网络，用容器名代替localhost       |
| 公网访问慢             | 升级ZeroNews 4.0 RC，带宽从2M提升到5M     |
| 数据持久化丢失         | 正确挂载数据卷，确保重启不丢失            |
